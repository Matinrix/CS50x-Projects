## 💜 About This Repository
Welcome to **Week 2** of Harvard’s _CS50_ course!  
This week dives deep into **arrays**, **strings**, and **character manipulation** in C.  
You’ll learn how characters map to numbers (ASCII), how to process text character by character, and implement algorithms for tasks like text analysis and scoring.

> 🧭 These exercises build foundational programming skills, crucial for understanding more complex data structures and algorithms.

---

## ⚙️ Tech Stack
- **Language:** C (CS50 standard syntax)  
- **Compiler:** GCC / clang  
- **Editor:** VS Code / CS50 IDE  
- **Encoding:** UTF‑8 (for emojis and Unicode visualization)

---

## 🧠 Concepts Covered
- Arrays: Declaration, initialization, and indexing  
- Strings: C-style strings (null-terminated character arrays)  
- Character handling: ASCII values, conversion to uppercase/lowercase  
- Loops: `for`, `while` for string and array traversal  
- Functions: Defining and calling custom functions (`getLen`, `compute_score`, `toUpper`)  
- Conditional logic: `if-else if-else` for comparisons  
- Basic algorithms: Text scoring, binary representation

## 🗂 Files Overview

Here are the source codes completed for this week 🔮  
Each implements a concept from the lectures and labs.

| 🗂 File Name      | 🎯 Title               | 📝 Description                                                                                                                                                                                                                                                                                       |
|-------------------|------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `Bulbs.c`         | **Binary Bulbs 💡**    | Takes user input and converts each character into its 8-bit binary representation using yellow (1) and black (0) emoji bulbs. A visual way to understand ASCII-to-binary conversion.                                                                                                                              |
| `Readability.c`   | **Readability Index 📖** | Calculates the readability of text using the *Coleman–Liau* formula (`index = 0.0588L - 0.296S - 15.8`), based on letter (`L`), word (`W`), and sentence (`S`) counts. It estimates the U.S. grade level needed to understand the text.                                                                    |
| `scrabble.c`      | **Scrabble Score 🅂**   | A two-player game where the score for each word is calculated based on Scrabble letter values. It declares winners based on the total score. _(Note: This version may have compilation errors as provided, like incorrect array declarations for `word1`/`word2` and potential issues in `toUpper`)._ |
| `scrabble2.c`     | **Scrabble Plus 🔤**  | An improved version of the Scrabble scoring game. It aims to fix compilation errors from the original (`scrabble.c`), potentially refining functions like `getLen` and `toUpper` for more robust character processing and scoring. It continues the two-player word-scoring logic.                    |
